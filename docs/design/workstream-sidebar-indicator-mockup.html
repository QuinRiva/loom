Workstream activity on the sidebar row

How an orchestrator row tells you, at a glance, the rolled-up state of its sub-thread graph and how many workers are live right now — without opening the workstream panel.

Rationale

Today a row carries one leading own-status pill (Working / Pending Approval / Awaiting Input / Plan Ready / Completed). That pill describes the orchestrator turn itself. The footgun: an orchestrator can be idle while its sub-threads grind away — the row looks finished when 37 workers are still running.

The fix is a separate trailing activity indicator that summarises the whole DAG, so the leading pill keeps meaning "this thread's turn" and the new glyph means "everything below it". One trailing pill carries both signals: colour/icon = rolled-up graph state, number = live worker count. A big count renders as a badge ("37"), never 37 dots.

Deadlocked

Colours reuse the existing board palette (running=sky, blocked/needs=amber, done=emerald, planned=slate). Deadlock is the one new colour — rose — because it is a danger state with no equivalent today and it deserves to stand out.

Same glyph, the count just grows. Caps at 99+ so the row never reflows. Coexists with the legacy terminal/cloud trailing icons.

Needs you

Nothing running; the graph is parked on a human gate. Amber + alert triangle; count is how many nodes await you (2 approvals here).

Deadlocked

Zero running, zero human gates, work still incomplete — a stuck DAG. Rose with a soft pulsing halo so it pulls your eye on an otherwise calm sidebar.

All done

Every node complete. Just a check, no number — the count would only ever be 0. Calm, recedes.

Leaf thread

No sub-threads → no indicator at all. Absence is the signal; we never show a "0" badge on a thread that has no workstream.

Hover — the breakdown behind the single count

The badge is a summary; hover (same Tooltip pattern as the existing trailing icons) expands the rolled-up number into its per-state parts.
